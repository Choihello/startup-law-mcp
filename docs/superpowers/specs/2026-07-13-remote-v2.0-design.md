# startup-law-mcp v2.0 설계 — 원격 MCP 서버 (Fly.io)

- 작성일: 2026-07-13
- 상태: 사용자 승인 완료
- 선행: v1.4 (main, 도구 13개, pytest 126개, 주간 자동 동기화 가동)

## 목표

koica-reg-mcp의 원격 배포 패턴: 설치 없이 커넥터 URL 하나(`https://<앱>.fly.dev/mcp`)로
Claude Desktop·claude.ai·Claude Code에서 연결. 주간 데이터 PR 머지 시 원격도 자동 최신화.

## 확정 결정

| 결정 | 선택 | 근거 |
|---|---|---|
| 플랫폼 | Fly.io (사용자 확정) | 레퍼런스 동일 패턴, auto-stop 절전으로 유휴 비용 ~0. 계정·카드는 사용자 직접 |
| 재배포 | main push 시 자동 (fly-deploy.yml, 코드·데이터 경로 필터) | 주간 PR 머지 → 원격 자동 최신화 |
| 원격 도구 표면 | 12개 (sync_programs 제외 — admin) | 원격에서 아무나 서버측 수집 유발 금지 (koica 동일). data_status는 읽기라 포함 |
| 시크릿 | 원격 서버 자체는 0개 | 읽기 전용 — 데이터는 이미지에 구움. 갱신은 재배포로 |
| 전송 | FastMCP streamable-http, stateless, /mcp, 0.0.0.0:8080 | MCP 표준 원격 전송 |
| 리전 | nrt(도쿄) | 한국 최저 지연 |

## 구성

1. **server.py**: `register_tools(mcp, include_admin: bool = True)` — sync_programs
   정의를 `if include_admin:` 게이트 안으로. 로컬 stdio는 13개 유지.
2. **server_http.py** (신규, 얇게): server의 SERVER_INSTRUCTIONS·register_tools를
   import, `include_admin=False`, streamable-http 실행.
3. **Dockerfile**: python:3.12-slim → 의존성 → 코드+data 복사 → **빌드 시
   `law_search.py build`** (인덱스를 이미지에 구움) → server_http 실행. PYTHONUTF8=1.
4. **fly.toml**: app명(전역 유일 — 배포 시 확정), nrt, internal_port 8080,
   auto_stop_machines/auto_start_machines, min_machines_running 0,
   shared-cpu-1x·512MB.
5. **fly-deploy.yml**: main push + 경로 필터(`**.py`, `data/**`, Dockerfile,
   fly.toml, 워크플로 자신) → `flyctl deploy --remote-only` (FLY_API_TOKEN),
   concurrency 그룹.
6. **scripts/remote_smoke.py**: streamable-http 클라이언트로 initialize →
   tools/list(12개 확인) → search_law 1회 호출 — 로컬 기동 검증과 원격 E2E
   양쪽에서 사용.
7. **README**: 최상단 "바로 쓰기 — 설치 불필요" 섹션 (커넥터 등록 표: Claude
   Desktop/claude.ai/Claude Code), 원격판 12개 도구·절전 첫 요청 지연 안내,
   로드맵 v2.0 완료. URL은 배포 후 실제 앱명으로 확정.

## 진행 순서

- T1(도구 게이트+HTTP 엔트리포인트)·T2(컨테이너·배포 설정+로컬 검증+README)는
  계정 없이 브랜치에서 진행.
- T3(실배포)는 사용자 액션 후: ① fly.io 가입+카드, ② `fly auth login`. 이후
  앱 생성·배포·토큰 발급(`fly tokens create deploy`)·`FLY_API_TOKEN` 시크릿
  등록·원격 smoke·main 병합(자동 배포 검증)은 컨트롤러 수행.
- **병합 순서 주의**: fly-deploy.yml이 main에 있으면 토큰 없을 때 실패 런이
  생기므로, v2.0 브랜치는 T3(토큰 등록)까지 마친 뒤 main에 병합한다.

## 검증

- 단위: register_tools 게이트 (로컬 13 / 원격 12, 차집합 == {sync_programs}).
- 로컬 E2E: `python server_http.py` 기동 → remote_smoke.py로 initialize·
  tools/list·search_law 실호출 (Docker 없이도 전송 계층 검증).
- (가능 시) Docker 로컬 빌드로 이미지 내 인덱스 생성 확인.
- 원격 E2E: 배포된 URL에 remote_smoke.py + Claude 클라이언트 연결은 사용자 확인.
