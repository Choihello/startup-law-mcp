## 설계 (제시됨, 사용자 승인 대기 상태)

### 1. 서버 도구 `match_programs` — 찾기·스크리닝

- 입력: 나이, 지역, 예비/기창업, 업력(년), 업종 키워드 — 전부 선택 인자
- 동작: 공고 `지원대상` 텍스트에서 조건을 정규식 사전으로 추출
  (만 39세 이하/청년, 업력 N년 이내, 예비창업자, 지역명, 여성 등) → 프로필 대조
- 출력: 탈락 사유 없는 공고 목록 + 조건별 판정 라벨, 마감순 정렬
  - `match` — 근거 문구 포함
  - `unknown` — 파싱 불가 → "원문 확인 필요" 정직 표시
- **원칙: 판정기가 아니라 스크리닝** — 잘못된 확신이 상담에서 가장 위험.
  기존 실재성 게이트 철학 유지. 표준 라이브러리만, pytest.

### 2. Claude Code 스킬 `startup-consult` — 해석·상담 양식

- 흐름: 대화에서 프로필 추출(빠진 건 한 번에 몰아 질문) → `match_programs` →
  상담 카드 출력: 사업명 · D-day · 충족 조건 · **확인 필요 조건** · 원문 링크
- 자격 해석: 애매 조건(업력 기산일, 공동대표, 폐업 이력 등)은 `search_law`·공고
  원문으로 **근거를 제시** (결론 단정 금지)
- `references/eligibility-notes.md`: 상담 중 발견한 해석 노하우 적립처.
  "이번에 알게 된 거 노트에 추가해줘" 한 마디로 축적 — 차별화 자산.

### 3. 배포·공개

- `match_programs` → 서버 병합 시 원격 12→13도구
- 스킬: `~/.claude/skills/` 설치 + repo `skills/startup-consult/` 공개
  (도메인 노하우의 프롬프트 자산화 — 포트폴리오 스토리)
- 컨벤션 유지: feat/v2.1 브랜치 · pytest · 병합 전 사용자 확인
